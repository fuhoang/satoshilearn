---
title: Hard Forks vs Soft Forks
---

Forks are changes to network rules. A soft fork tightens or narrows the accepted rules in a way that remains compatible with older software under certain conditions. A hard fork creates a rule change that is not backward compatible, meaning participants who do not upgrade may follow a different chain.

This sounds abstract, but the practical point is that not all network changes are equal. Some updates can be adopted without splitting the network. Others risk creating separate versions of the ledger if enough participants disagree.

In Bitcoin, rule changes matter because the network is decentralized. No one can simply announce a change and force everyone else to follow it. Users, miners, businesses, and node operators all play a role in whether a rule change gains real adoption.

This is one reason Bitcoin changes slowly. That slowness can be frustrating, but it is also protective. Monetary systems should not be easy to rewrite casually.

The beginner lesson is that forks are really about governance and compatibility. They show how decentralized networks evolve under shared rules rather than central command.
